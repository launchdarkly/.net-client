LaunchDarkly.Client Namespace

This is the main namespace for the LaunchDarkly .NET SDK. You will most often use LdClient (the SDK client), Configuration (configuration options for the client), and User (user properties for feature flag evaluation).
Classes
  ClassDescription
Components
Provides factories for the standard implementations of LaunchDarkly component interfaces.
Configuration
Configuration options for LdClient. This class should normally be constructed with Builder(String).
ConfigurationExtensions Obsolete.
Extension methods that can be called on a Configuration to add to its properties. It is preferable to use Builder(String) instead.
CustomEvent
An analytics event generated by the Track method. Used internally.
EvaluationDetailT
An object returned by the "variation detail" methods of the client, combining the result of a flag evaluation with an explanation of how it was calculated.
EvaluationReason
Describes the reason that a flag evaluation produced a particular value. Subclasses of EvaluationReason describe specific reasons.
EvaluationReasonError Obsolete.
Indicates that the flag could not be evaluated, e.g. because it does not exist or due to an unexpected error. In this case the result value will be the default value that the caller passed to the client.
EvaluationReasonFallthrough Obsolete.
Indicates that the flag was on but the user did not match any targets or rules.
EvaluationReasonOff Obsolete.
Indicates that the flag was off and therefore returned its configured off value.
EvaluationReasonPrerequisiteFailed Obsolete.
Indicates that the flag was considered off because it had at least one prerequisite flag that either was off or did not return the desired variation.
EvaluationReasonRuleMatch Obsolete.
Indicates that the flag was considered off because it had at least one prerequisite flag that either was off or did not return the desired variation.
EvaluationReasonTargetMatch Obsolete.
Indicates that the user key was specifically targeted for this flag.
Event
An analytics event that may be sent to LaunchDarkly. Used internally.
FeatureFlagsState
A snapshot of the state of all feature flags with regard to a specific user. See calling AllFlagsState(User, FlagsStateOption).
FeatureFlagsStateBuilder
A builder for constructing FeatureFlagsState instances.
FeatureRequestEvent
An analytics event generated by feature flag evaluation. Used internally.
FeatureStoreCacheConfig
Parameters that can be used for IFeatureStore database integrations that support local caching.
FlagsStateOption
Optional parameters that can be passed to AllFlagsState(User, FlagsStateOption).
IdentifyEvent
An analytics event generated by the Identify method. Used internally.
ILdClientExtensions
Convenience methods that extend the ILdClient interface.
InMemoryFeatureStore
In-memory, thread-safe implementation of IFeatureStore.
LdClient
A client for the LaunchDarkly API. Client instances are thread-safe. Applications should instantiate a single LdClient for the lifetime of their application.
LdValueArrayBuilder
An object returned by BuildArray for building an array of values.
LdValueConvert
Predefined instances of LdValueConverterT for commonly used types.
LdValueConverterT
Defines a conversion between LdValue and some other type.
LdValueObjectBuilder
An object returned by BuildObject for building an object from keys and values.
User
Attributes of a user for whom you are evaluating feature flags.
UserExtensions Obsolete.
Extension methods that can be called on a User to add to its properties. The preferred method is to use Builder(String) instead.
VersionedDataKindT
Objects used by IFeatureStore implementations to denote a specific collection of IVersionedData-derived objects.
